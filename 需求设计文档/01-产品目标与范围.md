# 产品目标与范围

## 目标用户与场景假设

- 室内设计师：需要快速出方案、出尺寸、出效果图前的结构底图
- 家装用户：在浏览器中完成基础户型绘制与功能布局
- 设计机构运营：需要结构化数据流向 3D 与渲染

## 核心问题

- 现有在线工具难以兼顾精确绘制与易用交互
- 2D 与 3D 之间的数据口径不一致，导致后续建模成本高
- 需要形成可迭代的引擎能力，从 2D 到 3D 到照片级渲染

## 产品愿景

- 以 2D 精确绘制为核心入口，逐步升级到 3D 与实时渲染
- 建立统一的场景数据模型，保证每个阶段复用
- 提供工程图风格与可视化风格两类输出

## 约束与边界

- 首阶段仅实现 Canvas 2D，不引入 WebGL
- 仅覆盖核心交互：墙体、门窗、测量、网格吸附
- 暂不支持多用户协作与云端渲染

## 成功指标

- 新用户 10 分钟内完成首个可量尺户型
- 2D 绘制平均学习成本低于 15 分钟
- 2D 图转 3D 几何失败率低于 3%
- 生成的 2D 图纸具备可输出与可复用性

## 数据模型统一策略

- 以场景图为核心：房间、墙体、门窗、标注与网格
- 2D 阶段输出结构化 JSON，作为后续 3D 的唯一输入

## 方案路径

- 阶段一：Canvas 2D 高精度绘制与编辑
- 阶段二：WebGL 3D 挤出与材质系统
- 阶段三：WebGPU 实时光影与仿真
